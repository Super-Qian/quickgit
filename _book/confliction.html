分支冲突

如果我们在newbranch分支上修改并提交了filename文件，切换到master分支后，又对filename进行了修改和提交，这时如果合并分支，就会出现分支冲突，导致合并失败，此时使用git status命令，返回信息中会有一句类似与both modified: filename的信息，这时可以直接查看filename文件，会发现如下形式：

This is a test file
hello
<<<<<<< HEAD
world
=======
git
>>>>>>> feature1

Git用<<<<<<<，=======，>>>>>>>标记出不同分支的内容，需要手动修改后再执行：

  1. git add filename
  2. git commit -m "comment"
  3. git merge newbranch

最后删除newbranch分支即可
